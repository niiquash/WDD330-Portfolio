JavaScript Grammar

  • JS syntax is know as C-style because each statement ends with a new line of semicolon.
  • Code blocks are placed in curly braces.
  • There are quite a number of reserved words. Even though 'undefined', 'Nan', and 'Infinity' are not reserved, they should be treated as such.

Primitive Data Types

This is a list of primitive data types in JavaScript. Any value that is not included in the list is an object:

  • String - "Hello"
  • Symbol - "const uniqueID = Symbol();"
  • Number - 157
  • Boolean - true/false
  • Undefined - undefined
  • Null - null

In JS, you can verify the type of value by using the typeof operator.

Variables

const and let are keywords used to declare variables. Const is used to declare immutable variables of primitive data types, while let is used to declare mutable variables. That said, if the variable holds an array, function or object, const will not make it immutable. Meaning that the object can still be mutated. What const does not allow, however, is for the variable to be assigned to another object, array, or function.

"let" and "const" have block scope, meaning that their values are only accessible within the block they are declared. It is generally bad practice to declare globally scoped variables in programming, as this can cause issues with reusing the same variable names in large programs, and having values overwrite each other.

Variable Naming

Constants and variable names can start with an upper/lower-case letter, underscore"_", or dollar sign "$". They can contain numbers, but cannot start with numbers. Examples: "$name", "_answer", "firstName", "address_line1".

String Properties and Methods

String variables have both properties and methods. The similarity between properties and methods is their use of the dot notation. An example of string property is the ".length" as in "name.length" property. An example of a method is ".toUpperCase()" as in "name.toUpperCase()". Methods contain parentheses, while properties do not.

Verifying if a Value is a Number

You can check if a value is a number than can be used by using the Number.isInfinite() methods If the return value is "true" it means the number is not "Infinity", "-Infinity", or "Nan".
